# 📖 使用指南

## 工作流

### 每周会议前（15 分钟）

1. 新建 `meetings/YYYY-MM-DD.md`（从模版复制）
2. 填写"本周完成"—— 对照上次 TODO 打勾
3. 把本周跑的实验结果整理成表格（CSV → 粘给 LLM 生成 Markdown table）

### 会议中

4. 填写"关键观察"和"讨论与指导"
5. 明确**一句话结论**和**下周 TODO**

### 会议后（5 分钟）

6. 更新 `README.md` 的两个索引表格（会议索引 + 实验索引）
7. 在上一次会议文件里填写"下次会议"链接

---

## 如何用 LLM 辅助分析

把以下内容打包发给 LLM，可以得到最具体的建议：

```
# 上下文
[粘贴 README.md 的"会议索引"表格]

# 最新实验
[粘贴最新 EXP-XXX.md 的"配置"和"结果"表格]

# 问题
根据以上历史 trace，我目前遇到了 [具体问题]，请给出具体建议。
```

**技巧**：表格用 CSV 格式存在注释里（见实验模版），随时可以复制出来直接粘给 LLM，不需要重新整理格式。

## 图片规范

- 文件名格式：`YYYY-MM-DD-描述.png` 或 `EXP-XXX-描述.png`
- 放在 `assets/` 目录
- **每张图必须有 caption**，格式：`*caption: 一句话说明图的核心信息*`
- Caption 要说明"这张图证明了什么"，而不是"这张图展示了什么"

**好 caption 示例**:  
✅ `*caption: EXP-003（蓝）过拟合程度比 EXP-002（橙）减少 38%，mixup 有效*`

**差 caption 示例**:  
❌ `*caption: 两组实验的训练曲线对比图*`

## 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 会议文件 | `YYYY-MM-DD.md` | `2024-03-15.md` |
| 实验文件 | `EXP-NNN.md` | `EXP-003.md` |
| 图片 | `EXP-NNN-描述.png`, 放置于assets文件夹| `EXP-003-loss-curve.png` |

---

## 最小可行记录

如果时间紧，这几项是**必须填**的：

- [ ] 会议：**一句话结论** + **下周 TODO**
- [ ] 实验：**假设** + **结论**（一句话）+ **结果表格**
- [ ] README：更新两个索引表格

## 📁 仓库结构

```
.
├── README.md              # 本文件，项目总览
├── meetings/              # 周会记录
│   ├── YYYY-MM-DD.md      # 每次会议一个文件
│   └── ...
├── experiments/           # 实验记录
│   ├── EXP-001.md
│   └── ...
└── assets/                # 图片等资源
    └── ...
```

## 💡 给 LLM 的上下文提示

> 当你需要 LLM 帮你分析时，可以把下面这段话 + 相关实验/会议文件一起发给它：

```
我在做 [研究方向] 相关研究。以下是我的实验记录和会议总结，
请根据这些历史 trace 给我具体的下一步建议。
[粘贴实验表格或会议摘要]
```
